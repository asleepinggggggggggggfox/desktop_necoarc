# 桌面宠物 AI 语音助手细化开发计划

## 1. 项目定位

制作一个透明悬浮的桌面宠物程序。角色显示在屏幕中间，左右两侧显示四个对齐的对话气泡：

- 左上：用户上一次输入
- 左下：用户本次输入
- 右上：AI 上一次输出
- 右下：AI 本次输出

用户通过麦克风说话，程序把语音发送到讯飞开放平台在线语音识别 API 转成文字，再发送给 DeepSeek API。DeepSeek 返回后，本轮输入和输出显示在下方两个气泡中；下一轮开始前，它们会滚动到上方的“上一次”气泡中。

第一版目标不是做复杂宠物系统，而是先完成一个稳定闭环：

`语音输入 -> 讯飞在线语音识别 -> DeepSeek 回复 -> 四气泡界面更新`

## 2. 效果图排版理解

效果图只作为粗略布局参考，最终界面要比效果图更规整。视觉核心是“三栏对齐布局”：

- 中间是角色立绘，是视觉中心，占据最大高度。
- 左侧是用户气泡，两枚上下排列。
- 右侧是 AI 气泡，两枚上下排列。
- 左右气泡必须在横向行高上对齐，形成清楚的上排和下排。
- 四个气泡围绕角色展开，但不压住角色主体。
- 整体背景透明，像贴在桌面上的悬浮小组件。

### 2.1 画布比例

效果图接近横向矩形，建议第一版窗口采用固定初始尺寸：

- 初始窗口宽度：`1000px`
- 初始窗口高度：`520px`
- 透明背景
- 无系统标题栏
- 始终置顶
- 可拖动整个窗口

窗口内容按三列组织，但气泡不跟随文本内容改变外框尺寸：

- 左侧气泡区：约 `32%`
- 中间角色区：约 `36%`
- 右侧气泡区：约 `32%`
- 上排气泡共用同一个 `y` 坐标
- 下排气泡共用同一个 `y` 坐标
- 左右气泡共用同一套宽高

### 2.2 角色位置

角色立绘位于窗口水平居中位置，底部接近窗口下边缘。

建议规则：

- 角色图片使用透明 PNG。
- 角色高度约为窗口高度的 `80% - 90%`。
- 角色水平居中。
- 角色底部留 `30px - 45px` 空白，避免贴边。
- 角色层级低于气泡，气泡可在视觉上靠近角色，但不要遮挡脸部和主体手势。
- 角色左右两侧至少给气泡尾巴留出 `18px - 28px` 间隔，避免尾巴贴到角色身上。

### 2.3 左侧用户气泡

左侧两个气泡纵向排列：

- 左上气泡：用户上一次输入
- 左下气泡：用户本次输入

排版规则：

- 气泡靠左，但不要贴窗口边缘。
- 左边距建议 `70px - 95px`。
- 上气泡顶部距窗口顶部约 `65px - 80px`。
- 下气泡顶部距窗口顶部约 `295px - 315px`。
- 两个气泡宽高一致，形成稳定节奏。
- 气泡尾巴朝向中间角色，左侧气泡尾巴位于右下方。
- 左侧两个气泡的左边缘必须完全对齐。

### 2.4 右侧 AI 气泡

右侧两个气泡纵向排列：

- 右上气泡：AI 上一次输出
- 右下气泡：AI 本次输出

排版规则：

- 气泡靠右，但不要贴窗口边缘。
- 右边距建议 `90px - 115px`。
- 上气泡顶部与左上气泡完全对齐。
- 下气泡顶部与左下气泡完全对齐。
- 右侧两个气泡的右边缘必须完全对齐。
- 气泡尾巴朝向中间角色，右侧气泡尾巴位于左下方。

### 2.5 气泡视觉样式

效果图里的气泡是手绘感较强的白底黑边气泡。第一版可以先用规则图形实现，后续再增强手绘感。

建议样式：

- 背景色：白色或接近白色
- 边框：黑色，`4px`
- 圆角：`26px - 32px`
- 气泡主体尺寸：建议 `220px x 118px`
- 气泡最小尺寸：`200px x 104px`
- 气泡最大尺寸：`250px x 132px`
- 尾巴尺寸：约 `45px x 28px`
- 字体颜色：深灰或黑色
- 字号：`15px - 17px`
- 文本居中显示
- 长文本自动换行
- 单个气泡最多显示 `4` 行正文
- 超长文本优先省略，必要时再把字号降到 `14px`
- 四个气泡尺寸必须一致，不允许因为文本长短导致外框变大或变小

### 2.6 对齐规则

四个对话框要按规则对齐，而不是按效果图的手绘位置随意摆放。

必须满足：

- 左上气泡和右上气泡：顶部对齐、尺寸一致。
- 左下气泡和右下气泡：顶部对齐、尺寸一致。
- 左侧两枚气泡：左边缘对齐。
- 右侧两枚气泡：右边缘对齐。
- 上下两排之间的垂直间距一致，建议 `105px - 130px`。
- 左右两列到角色中心的视觉距离接近一致。
- 所有气泡固定尺寸，不随内容变化。
- 尾巴可以伸出主体框，但不能参与对齐计算。

### 2.7 层级关系

推荐层级从低到高：

1. 透明窗口背景
2. 角色立绘
3. 四个气泡
4. 状态提示、右键菜单、调试控件

气泡和角色之间要留出呼吸空间。气泡不应挡住角色脸、手和身体主要轮廓。

## 3. 界面状态设计

界面长期展示四个气泡，但不同阶段内容不同。

### 3.1 初始状态

- 用户上一次输入：`用户上一次输入`
- 用户本次输入：`按住快捷键开始说话`
- AI 上一次输出：`AI 输出（上次）`
- AI 本次输出：`等待你的问题`

### 3.2 录音中

- 用户本次输入显示：`正在录音...`
- AI 本次输出保持不变
- 可选：角色切换为倾听表情

### 3.3 识别中

- 用户本次输入显示：`正在识别...`
- AI 本次输出保持不变

### 3.4 AI 思考中

- 用户本次输入显示识别结果
- AI 本次输出显示：`思考中...`
- 可选：角色切换为思考表情

### 3.5 回复完成

- 用户本次输入显示本轮语音识别文本
- AI 本次输出显示 AI 回复
- 内存中保存本轮输入和输出

下一轮开始时：

- 上一轮“用户本次输入”移动到“用户上一次输入”
- 上一轮“AI 本次输出”移动到“AI 上一次输出”
- 下方两个气泡清空并进入新一轮状态

## 4. 核心功能范围

第一版必须完成：

- 透明无边框桌面窗口
- 窗口始终置顶
- 角色透明 PNG 显示
- 四个固定位置气泡
- 气泡文本自动换行
- 鼠标拖动窗口
- 右键菜单退出程序
- 快捷键按住录音、松开停止
- 讯飞开放平台在线语音识别
- DeepSeek 对话调用
- 上一轮输入和输出保存
- 基础错误提示

第一版暂不强求：

- 多角色切换
- 复杂动画
- 本地大模型

- 开机自启动
- 语音朗读

## 5. 技术方案

### 5.1 开发语言

使用 Python。

理由：

- 桌面 GUI 原型开发快
- 调用麦克风、网络 API 和本地文件方便
- 适合先做可运行 Demo

### 5.2 GUI 框架

使用 PySide6。

需要用到的能力：

- `Qt.FramelessWindowHint` 实现无边框
- `Qt.WindowStaysOnTopHint` 实现置顶
- `Qt.WA_TranslucentBackground` 实现透明窗口
- `QLabel` 或自定义 `QWidget` 显示角色图片
- 自定义 `BubbleWidget` 绘制气泡和尾巴
- 鼠标事件实现窗口拖动
- `QMenu` 实现右键菜单

### 5.3 录音

使用 `sounddevice` 录音，使用 `soundfile` 保存临时音频。

第一版交互：

- 按住快捷键开始录音
- 松开快捷键停止录音
- 录音保存为临时 wav 文件
- 录音结束后自动进入语音识别

### 5.4 语音识别

使用讯飞开放平台在线语音识别 API。

第一版建议：

- 语音识别服务使用讯飞开放平台在线 API
- 默认识别语言为中文普通话
- 录音结束后把音频按讯飞接口要求上传或通过 WebSocket 发送
- 按讯飞接口要求处理鉴权参数、音频格式、采样率和分片发送
- 网络请求放到后台线程，避免卡住 GUI

### 5.5 DeepSeek 对话

使用 DeepSeek API 生成 AI 回复。

处理流程：

- 将识别出的文本作为用户输入
- 按 DeepSeek Chat Completions 兼容接口格式发送请求
- 获取 DeepSeek 返回的回复文本
- 更新右下气泡
- 保存本轮记录

### 5.6 配置管理

配置内容建议放在 `.env` 和 `config.yaml` 中。

`.env`：

- `DEEPSEEK_API_KEY`
- `XUNFEI_APP_ID`
- `XUNFEI_API_KEY`
- `XUNFEI_API_SECRET`
- 其他第三方 API Key

`config.yaml`：

- DeepSeek 模型名
- DeepSeek API Base URL，默认 `https://api.deepseek.com`
- 讯飞语音识别接口地址
- 讯飞语音识别语言，默认中文普通话
- 音频采样率，建议第一版固定为 `16000`
- 音频编码格式，按讯飞接口要求配置
- 角色图片路径
- 快捷键
- 窗口初始位置
- 窗口初始尺寸
- 是否始终置顶
- 临时音频目录

## 7. 模块职责

### 7.1 `main.py`

- 加载配置
- 初始化 Qt 应用
- 创建主窗口
- 绑定核心服务
- 启动事件循环

### 7.2 `ui/main_window.py`

- 创建透明无边框窗口
- 布局角色和四个气泡
- 处理拖动、置顶、右键菜单
- 接收业务状态并刷新界面

### 7.3 `ui/bubble_widget.py`

- 绘制白底黑边气泡
- 根据左右方向绘制不同尾巴
- 控制文本换行、居中、省略
- 暴露 `set_text()` 方法

### 7.4 `ui/character_widget.py`

- 加载角色 PNG
- 按窗口高度自适应缩放
- 后续可扩展不同表情和动作

### 7.5 `core/audio_recorder.py`

- 控制录音开始和结束
- 保存临时音频文件
- 处理麦克风不可用、空录音等问题

### 7.6 `core/xunfei_speech_to_text.py`

- 按讯飞开放平台要求完成鉴权
- 上传音频或建立 WebSocket 识别连接
- 发送音频数据
- 获取识别文本
- 处理讯飞错误码、API 错误和网络超时

### 7.7 `core/deepseek_client.py`

- 发送用户文本
- 调用 DeepSeek API
- 获取 DeepSeek 回复
- 处理 API 错误和网络超时

### 7.8 `core/conversation_state.py`

- 保存四个气泡对应的文本
- 在新一轮开始时移动上一轮内容
- 提供统一状态给 UI

## 8. 开发阶段

### 阶段一：静态界面原型

目标：先复现效果图排版。

任务：

- 创建透明窗口
- 显示角色图片
- 绘制四个气泡
- 调整气泡位置、尺寸、尾巴方向
- 实现窗口拖动和右键退出
- 使用假文本验证长短文本展示

验收标准：

- 打开程序后，视觉结构与效果图一致
- 中间角色清晰，四个气泡分布在左右
- 背景透明，无标题栏
- 气泡文字不会溢出边界

### 阶段二：文本对话闭环

目标：不用语音，先验证 DeepSeek 对话和气泡状态。

任务：

- 增加临时文本输入入口
- 调用 DeepSeek API
- 更新右下 AI 气泡
- 实现上一轮记录滚动
- 加入 AI 请求中的状态提示

验收标准：

- 输入一句话后，AI 能返回结果
- 本轮输入和输出显示在下方
- 下一次输入开始时，上轮内容移动到上方

### 阶段三：接入语音识别

目标：完成真实语音输入，并通过讯飞开放平台完成在线识别。

任务：

- 接入麦克风录音
- 接入快捷键录音控制
- 保存临时音频
- 调用讯飞开放平台在线语音识别 API
- 将识别文本送入 DeepSeek 对话流程

验收标准：

- 按住快捷键说话，松开后自动识别
- 讯飞返回的识别结果显示在左下气泡
- AI 回复显示在右下气泡

### 阶段四：体验优化

目标：让 Demo 更像可日常使用的小工具。

任务：

- 增加更清楚的错误提示
- 增加配置文件
- 增加托盘图标
- 增加角色状态图片
- 增加简单待机动画
- 保存窗口上次位置

验收标准：

- 网络、麦克风、API Key 出错时不会卡死
- 用户能通过配置调整模型、快捷键和图片路径
- 程序关闭再打开能恢复窗口位置

## 9. 错误处理

需要处理的情况：

- 麦克风不可用：提示 `无法使用麦克风，请检查权限或设备`
- 录音为空：提示 `没有检测到有效语音，请重试`
- 语音识别失败：提示 `语音识别失败，请检查网络或讯飞 API 配置`
- AI 回复失败：提示 `AI 回复失败，请检查网络或模型配置`
- 网络超时：提示 `请求超时，请稍后重试`
- 角色图片缺失：显示默认占位图或错误提示

所有网络请求都应设置超时时间，并放在后台线程中执行，避免界面无响应。

## 10. 排版实现建议

第一版不要依赖复杂响应式布局，可以使用相对坐标计算。效果图只提供大方向，实际实现优先保证对齐、尺寸合适、文字可读。

推荐坐标逻辑：

- 窗口宽 `W`，高 `H`
- 角色中心点：`W * 0.5`
- 角色底部：`H - 35`
- 气泡宽：`W * 0.22`，最小 `200`，最大 `250`
- 气泡高：`H * 0.23`，最小 `104`，最大 `132`
- 左气泡 x：`W * 0.08`
- 右气泡 x：`W - left_x - bubble_width`
- 上排气泡 y：`H * 0.13`
- 下排气泡 y：`H * 0.57`

四个气泡坐标应由同一组变量计算：

```text
left_x = W * 0.08
right_x = W - left_x - bubble_width
top_y = H * 0.13
bottom_y = H * 0.57

user_prev = (left_x, top_y)
user_current = (left_x, bottom_y)
ai_prev = (right_x, top_y)
ai_current = (right_x, bottom_y)
```

这样窗口缩放时仍能保持效果图的整体关系。

对齐验收标准：

- 上排两个气泡的 `y` 值完全一致。
- 下排两个气泡的 `y` 值完全一致。
- 左列两个气泡的 `x` 值完全一致。
- 右列两个气泡的 `x + width` 值完全一致。
- 四个气泡的 `width` 和 `height` 完全一致。
- 窗口缩放后仍保持上述关系。
