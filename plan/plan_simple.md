# 桌面宠物 AI 语音助手技术方案

## 1. 项目目标

制作一个桌面宠物程序，宠物显示在电脑桌面上，用户通过麦克风说话，程序把语音转换成文字，再把文字发送给 AI。AI 返回回答后，在桌面宠物旁边用文字气泡显示出来。

界面主要包括四部分内容：用户上一次输入、用户本次输入、AI 上一次输出、AI 本次输出。中间显示桌面宠物角色图片，整体效果类似透明悬浮窗。

---

## 2. 基本功能

第一版需要实现以下功能：

1. 桌面宠物显示在屏幕上，并且窗口透明、无边框、始终置顶。
2. 用户可以通过麦克风输入语音。
3. 程序调用在线语音识别 API，把语音转换为文字。
4. 程序把识别出来的文字发送给 AI 对话 API。
5. AI 返回回答后，在右侧气泡中显示文字。
6. 程序保存上一轮输入和输出，方便界面显示“上一次内容”。
7. 宠物窗口可以拖动，也可以通过右键菜单退出程序。

---

## 3. 技术选择

### 3.1 开发语言

使用 Python 开发。

Python 调用桌面窗口、麦克风、语音识别 API 和 AI API 都比较方便，适合快速做出原型。

### 3.2 桌面界面

使用 PySide6 制作桌面窗口。

PySide6 可以实现透明窗口、无边框窗口、窗口置顶、鼠标拖动、右键菜单和图片显示，适合制作桌面宠物。

### 3.3 麦克风录音

使用 sounddevice 和 soundfile 完成录音。

第一版采用按键录音方式，例如按下快捷键开始录音，松开后停止录音。这样实现简单，也能避免程序一直监听麦克风。

### 3.4 在线语音识别

使用在线语音识别 API，例如 OpenAI Whisper API 或其他语音识别服务。

程序把录好的音频文件上传给语音识别 API，API 返回识别出的文字。第一版优先使用在线识别，因为配置简单、识别准确率较高、本地电脑压力小。

### 3.5 AI 对话

使用 AI 对话 API 生成回答。

程序把语音识别得到的文字作为用户输入，发送给 AI。AI 返回回答后，程序把回答显示在桌面宠物右侧的气泡里。

---

## 4. 工作流程

程序运行后的流程如下：

1. 启动桌面宠物程序。
2. 加载宠物图片和四个文字气泡。
3. 用户按下录音快捷键。
4. 程序开始录音，界面显示“正在录音”。
5. 用户松开快捷键，程序停止录音。
6. 程序把录音保存为临时音频文件。
7. 程序调用在线语音识别 API。
8. API 返回识别出的文字。
9. 用户本次输入气泡显示识别结果。
10. 程序把这段文字发送给 AI。
11. AI 返回回答。
12. AI 本次输出气泡显示回答内容。
13. 本次输入和输出保存为上一轮记录。
14. 程序等待下一次语音输入。

---

## 5. 界面设计

桌面宠物界面分为中间、左侧和右侧三部分。

中间显示宠物角色图片，图片使用透明背景 PNG。左侧显示用户输入，右侧显示 AI 输出。上方气泡显示上一次内容，下方气泡显示本次内容。

气泡样式使用白色背景、黑色边框、圆角矩形，并带有小尾巴。文字过长时自动换行，避免超出气泡范围。

窗口整体采用透明背景，隐藏标题栏，并保持在桌面最上层。用户可以拖动宠物改变位置。

---

## 6. 状态提示

为了避免用户感觉程序卡住，界面需要显示不同状态。

录音时，用户本次输入气泡显示“正在录音……”。

录音结束后，用户本次输入气泡显示“正在识别……”。

识别成功后，用户本次输入气泡显示识别出的文字。

发送给 AI 后，AI 本次输出气泡显示“思考中……”。

AI 返回结果后，AI 本次输出气泡显示 AI 的回答。

如果识别失败，气泡显示“语音识别失败，请重试”。如果 AI 调用失败，气泡显示“AI 回复失败，请检查网络或 API Key”。

---

## 7. 项目结构

项目可以按下面方式划分：

main.py 作为程序入口，负责启动整个程序。

ui 文件夹负责桌面宠物窗口、气泡显示、图片显示和右键菜单。

core 文件夹负责录音、语音识别、AI 对话和对话记录。

assets 文件夹存放宠物图片和图标。

config 文件夹存放普通配置，例如模型名称、快捷键、图片路径等。

.env 文件保存 API Key，不要上传到 GitHub。

---

## 8. 配置内容

程序需要准备以下配置：

1. AI API Key。
2. 语音识别模型名称。
3. AI 对话模型名称。
4. 宠物图片路径。
5. 录音快捷键。
6. 是否窗口置顶。
7. 临时音频文件保存路径。
8. 语音识别语言，第一版设置为中文。

API Key 应该放在 .env 文件或环境变量中，不要直接写在代码里。

---

## 9. 错误处理

程序需要处理一些常见问题。

如果麦克风不可用，需要提示用户检查麦克风权限或设备连接。

如果录音内容为空，需要提示用户重新录音。

如果语音识别失败，需要提示用户检查网络、API Key 或账号额度。

如果 AI 回复失败，需要提示用户检查网络或 API 设置。

如果网络超时，程序不要卡死，可以显示失败提示，让用户重新尝试。

---

## 10. 开发阶段

### 第一阶段：界面原型

先完成桌面宠物窗口、宠物图片、四个气泡和拖动功能。这个阶段可以先用手动输入文字代替语音输入，主要验证界面效果。

### 第二阶段：接入 AI 对话

加入 AI 对话 API，让用户输入文字后，AI 能返回回答，并显示到气泡里。

### 第三阶段：接入语音识别

加入麦克风录音和在线语音识别 API。用户说话后，程序能自动转成文字，再发送给 AI。

### 第四阶段：优化体验

加入右键菜单、状态提示、异常处理、开机自启动、角色图片切换和简单动画。

---

## 11. 后期扩展

后期可以继续加入 AI 语音播报，让宠物不只显示文字，还能把回答读出来。

也可以加入多张宠物图片，实现待机、说话、思考、开心等不同状态。

还可以加入本地语音识别、本地大模型、聊天记录保存、系统托盘、桌面边缘吸附等功能。

---

## 12. 总结

这个项目的核心技术路线是：PySide6 桌面宠物界面 + 麦克风录音 + 在线语音识别 API + AI 对话 API + 气泡文字显示。

第一版不要做得太复杂，先完成“说话 → 转文字 → AI 回复 → 气泡显示”这个完整流程。等基础功能稳定后，再逐步加入动画、语音播报、角色切换和更多交互功能。
